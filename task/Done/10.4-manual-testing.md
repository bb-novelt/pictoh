### Observations / Issues Found

1. **Picture images show broken** ("?" placeholder) — in dev mode the built-in image paths (e.g. `/pictures/manger.webp`) resolve to 404. This is expected in dev mode and works in the production build where the images are pre-cached by the service worker.
2. **Navigation via square click is touch-only** — clicking squares with a mouse in non-edit mode does not navigate (onClick is intentionally `undefined` in normal mode; only `onTouchStart` fires). This is correct tablet-first behavior.
3. **Picture usage `lastUsedTime` is in-memory only** — it is not persisted to localStorage across reloads. The `AppConfig` type does not include a `pictureUsage` field. This is a known gap (PictureService comment mentions "persistence will be added in Epic 9") but Epic 9 storage was completed without addressing this. Not a blocker.

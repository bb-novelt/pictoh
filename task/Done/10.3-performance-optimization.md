# Task 10.3: Performance Optimization

**Epic**: Epic 10: Polish & Testing

## Task Details

- [x] Optimize re-renders with React.memo
- [x] Optimize image loading and caching
- [x] Minimize bundle size
- [x] All resources loaded on first access (no lazy loading)
- [x] Profile and optimize slow operations

## Completion Summary

- **`Square` wrapped with `React.memo`**: The 24-instance grid component now skips re-renders when its `square` data, `isEditMode`, and `onSquareClick` props are unchanged (e.g. when a different square is edited). Prop renamed from `onClick: () => void` to `onSquareClick: (square: SquareType) => void`.
- **Stable `handleSquareClick` in `Grid`**: Uses `useCallback` with `[isEditMode, handleInteraction, setEditingPosition]` so the callback reference is preserved when the store updates without changing `isEditMode`. `isEditMode` is extracted from the Valtio snapshot to a local boolean for compatibility with the React Compiler lint rule.
- **Stable `handleInteraction` in `useSquareInteraction`**: Wrapped with `useCallback(fn, [])` — no reactive dependencies since it only uses refs and module-level stable functions. This makes it a reliable stable dependency for Grid's memoization.
- **`PictureThumbnail` wrapped with `React.memo`**: Each thumbnail in the picture-selector grid skips re-renders unless its `picture`, `isSelected`, or `onSelect` props change.
- **Stable `handleSelect` in `PictureSelector`**: Uses `useCallback([onSelect, setRevision])` so the callback passed to all `PictureThumbnail` instances stays the same reference between renders triggered by search-text changes.
- **`decoding="async"` on images**: Added to `<img>` elements in `Square.tsx` and `PictureSelector.tsx` so the browser decodes images off the main thread, preventing UI jank.
- **Bundle cleanup**: Removed `react-router-dom` from the `manualChunks` vendor config in `vite.config.mts` since it is not used anywhere in the application.
- No lazy loading introduced — all code modules are bundled eagerly as required.
